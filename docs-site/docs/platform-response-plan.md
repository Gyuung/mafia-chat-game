# 채팅형 응답 설계

웹 게임의 규칙과 결과를 나중에 채팅형 인터페이스에서도 재사용할 수 있도록 응답 단위를 작게 유지합니다.

## 기본 명령 흐름

- 게임 시작: 참가자 수와 내 이름을 정합니다.
- 밤 행동: 역할에 맞는 대상 선택지를 보여줍니다.
- 낮 심문: 한 명을 선택하면 짧은 답변과 의심 단서를 반환합니다.
- 투표: 생존자 목록에서 제거 대상을 선택합니다.
- 결과: 승리 팀, 내 역할, XP, 레벨 변화를 요약합니다.

## 결과 카드에 포함할 정보

- 승리 팀
- 내 역할
- 생존 여부
- 획득 XP
- 레벨 진행도
- 새 칭호
- 핵심 사건 한 줄 요약

## 주의점

- 외부 플랫폼명을 코드와 문서의 핵심 제품명으로 고정하지 않습니다.
- 웹 UI와 채팅형 응답은 같은 게임 상태를 다른 표현 방식으로 보여주는 구조로 유지합니다.
- 플랫폼 정책 문구는 별도 운영 문서에서 관리하고, 게임 본문에는 노출하지 않습니다.
